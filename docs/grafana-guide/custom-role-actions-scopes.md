# RBAC 權限、動作和範圍

權限由動作和範圍組成。建立自訂角色時，請考慮使用者可以執行的動作，以及他們可以在哪些資源上執行這些動作。

- **動作：** 動作描述使用者可以在資源上執行哪些任務。
- **範圍：** 範圍描述可以在哪裡執行動作，例如讀取特定使用者設定檔。在此範例中，權限與範圍 `users:<userId>` 相關聯，以授予相關角色。

## 範圍定義

以下清單包含基於角色的存取控制範圍。

<!-- prettier-ignore-start -->
| 範圍                                                                 | 描述                                                                                                                                                                                                                                              |
| -------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <ul><li>`annotations:*`</li><li>`annotations:type:*`</li></ul>       | 將動作限制在一組註釋上。例如，`annotations:*` 匹配任何註釋，`annotations:type:dashboard` 匹配與儀表板關聯的註釋，而 `annotations:type:organization` 匹配組織註釋。 |
| <ul><li>`dashboards:*`</li><li>`dashboards:uid:*`</li></ul>          | 將動作限制在一組儀表板上。例如，`dashboards:*` 匹配任何儀表板，而 `dashboards:uid:1` 匹配 UID 為 `1` 的儀表板。                                                                                       |
| <ul><li>`datasources:*`</li><li>`datasources:uid:*`</li></ul>        | 將動作限制在一組資料來源上。例如，`datasources:*` 匹配任何資料來源，而 `datasources:uid:1` 匹配 UID 為 `1` 的資料來源。                                                                               |
| <ul><li>`folders:*`</li><li>`folders:uid:*`</li></ul>                | 將動作限制在一組資料夾上。例如，`folders:*` 匹配任何資料夾，而 `folders:uid:1` 匹配 UID 為 `1` 的資料夾。請注意，授予資料夾的權限會向下級聯到位於其下的子資料夾。               |
| <ul><li>`global.users:*`</li><li>`global.users:id:*`</li></ul>       | 將動作限制在一組全域使用者上。例如，`global.users:*` 匹配任何使用者，而 `global.users:id:1` 匹配 ID 為 `1` 的使用者。                                                                                              |
| <ul><li>`library.panels:*`</li><li>`library.panels:uid:*`</li></ul>  | 將動作限制在一組程式庫面板上。例如，`library.panels:*` 匹配任何程式庫面板，而 `library.panel:uid:1` 匹配 UID 為 `1` 的程式庫面板。                                                                    |
| <ul><li>`orgs:*`</li><li>`orgs:id:*`</li></ul>                       | 將動作限制在一組組織上。例如，`orgs:*` 匹配任何組織，而 `orgs:id:1` 匹配 ID 為 `1` 的組織。                                                                                             |
| <ul><li>`permissions:type:delegate`</li><ul>                         | 此範圍僅適用於與存取控制本身關聯的角色，並表示您只能透過建立新角色或進行指派來委派您的權限，或其子集。                                     |
| <ul><li>`permissions:type:escalate`</li><ul>                         | 需要此範圍來觸發基本角色權限的重設。它表示使用者可能會獲得他們之前沒有的額外權限。                                                                                  |
| <ul><li>`plugins:*`</li><li>`plugins:id:*`</li></ul>                 | 將動作限制在一組外掛程式上。例如，`plugins:id:grafana-oncall-app` 匹配 Grafana OnCall 外掛程式，而 `plugins:*` 匹配所有外掛程式。                                                                                           |
| <ul><li>`provisioners:*`</li><ul>                                    | 將動作限制在一組佈建程式上。例如，`provisioners:*` 匹配任何佈建程式，而 `provisioners:accesscontrol` 匹配基於角色的存取控制 [佈建程式](ref:rbac-grafana-provisioning)。           |
| <ul><li>`reports:*`</li><li>`reports:id:*`</li></ul>                 | 將動作限制在一組報告上。例如，`reports:*` 匹配任何報告，而 `reports:id:1` 匹配 ID 為 `1` 的報告。                                                                                                         |
| <ul><li>`roles:*`</li><li>`roles:uid:*`</li></ul>                    | 將動作限制在一組角色上。例如，`roles:*` 匹配任何角色，而 `roles:uid:randomuid` 只匹配 UID 為 `randomuid` 的角色。                                                                                            |
| <ul><li>`services:accesscontrol`</li><ul>                            | 將動作限制為僅以基於角色的存取控制服務為目標。您可以將此與 `status:accesscontrol` 動作結合使用。                                                                                                  |
| <ul><li>`serviceaccounts:*`</li><li>`serviceaccounts:id:*`</li></ul> | 將動作限制在組織中的一組服務帳戶上。例如，`serviceaccounts:*` 匹配任何服務帳戶，而 `serviceaccount:id:1` 匹配 ID 為 `1` 的服務帳戶。                                           |
| <ul><li>`settings:*`</li><ul>                                        | 將動作限制在設定子集上。例如，`settings:*` 匹配所有設定，`settings:auth.saml:*` 匹配所有 SAML 設定，而 `settings:auth.saml:enabled` 匹配 SAML 設定上的啟用屬性。                   |
| <ul><li>`teams:*`</li><li>`teams:id:*`</li></ul>                     | 將動作限制在組織中的一組團隊上。例如，`teams:*` 匹配任何團隊，而 `teams:id:1` 匹配 ID 為 `1` 的團隊。                                                                                              |
| <ul><li>`users:*`</li><li>`users:id:*`</li></ul>                     | 將動作限制在組織中的一組使用者上。例如，`users:*` 匹配任何使用者，而 `users:id:1` 匹配 ID 為 `1` 的使用者。                                                                                              |
| <ul><li>None</li><ul>                                                | If an action has "None" specified for the scope, then the action doesn't require a scope. For example, the `teams:create` action doesn't require a scope and allows users to create teams.                                                         |
